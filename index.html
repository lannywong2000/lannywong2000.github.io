Fully Convolutional Networks

FCN v.s. CNN

  • CNN requires fixed-size samples for training and predicting, and only predicts the whole picture as one object.

  • On the other hand, FCN replaces the last FCNN layer of CNN by a convolutional layer, and realizes semantic segmentation by classifying every pixel of the image as a type of object.

Restrictions of the traditional CNN

  1. Sliding window algorithm takes up a lot of storage space.
  2. The efficiency of algorithm is low due to duplicate neighboring pixels.
  3. The sensational area is restricted by the size of sliding windows.

OPC UA: Client Server Communication

This series are learning notes of the BRILLIANT videos Fundamentals of OPC UA by Industry40tv on YouTube.

OPC UA Service-Oriented Architecture

  1. Each component is providing a service.
  2. A component is exposing a list of standardized service interfaces.

Closed Loop Control using OPC UA

Levels of OPC UA Implementation

Request and Response

OPC UA Request-Response Model

  1. OPC UA client sends request to OPC UA server, which sends back response.
  2. It does NOT matter what language is used to implemented the OPC UA Stack on, the only things OPC UA client and server do is to encode and decode message into standard TCP protocol.
  3. OPC UA is optimized for bulk data transfer.

OPC UA Request and Response Headers

Establishing a Stateful OPC UA Client-Server Connection

The Transport Channel and Secure Channel are completely handled by OPC UA Stack. There are also Stateless OPC UA implements.

OPC UA Services

There are 37 services in the OPC UA service sets.

1. Discovery Service Set

  • FindServers Service
  • FindServersOnNetwork Service
  • GetEndpoints Service
  • RegisterServer Service
  • RegisterServer2 Service

2. SecureChannel Service Set

Create a secure channel, implemented on OPC UA Stack.

  • OpenSecureChannel Service
  • CloseSecureChannel Service

3. Session Service Set

Create a session channel to establish rules of communication between clients and servers at OPC UA application level.

  • CreateSession Service
  • ActivateSession Service
  • CloseSession Service
  • CancelSession Service

4. NodeManagement Service Set

For clients to add or delete nodes and references on OPC UA server address space.

  • AddNodes Service
  • AddReferences Service
  • DeleteNodes Service
  • DeleteReferences Service

5. View Service Set

For clients yo navigate through OPC UA server address space and consume information.

  • Browse Service
  • BrowseNext Service
  • TranslateBrowsePathsToNodeIds Service
  • RegisterNodes Service
  • UnregisterNodes Service

6. Query Service Set

Allow clients to issue queries to servers to access stored data.

  • QueryFirst Service
  • QueryNext Service

7. Attribute Service Set

Enable clients to access attributes of nodes for reading and writing purposes.

  • Read Service
  • Write Service
  • HistoryRead Service
  • HistoryUpdate Service

8. Method Service Set

For clients to invoke exposed methods

  • Call Service

9. MonitoredItem Service Set

Define services that allow clients to create, modify and delete monitored items used to monitor attributes for value changes and objects for events.

  • CreateMonitoredItems Service
  • ModifyMonitoredItems Service
  • SetMonitoringMode Service
  • SetTriggering Service
  • DeleteMonitoredItems Service

10. Subscription Service Set

In OPC UA, subscriptions are entities that issue notification messages periodically for the monitoring items assigned to them.

  • CreateSubscription Service
  • ModifySubscription Service
  • SetPublishingMode Service
  • Publish Service
  • Republish Service
  • TransferSubscription Service
  • DeleteSubscription Service

OPC UA: Information Model

This series are learning notes of the BRILLIANT videos Fundamentals of OPC UA by Industry40tv on YouTube.

Why Information Model?

Information modeling is key for Smart Manufacturing for components to communicate with each others directly instead of hierarchically. This requires consistency in information representation.

Representing Complex Model

  1. Base Node Class: basic unit of information.

    • Unique Node ID
      • (index of) namespace uri
      • identifier data type
      • identifier
    • Display Name
    • Node Class
    • Browse Name
    • Description
    • Actual Data Stored
    • References to other Nodes

  2. Node Classes Inherits from the Basic Node Class

    • Variable Node Class
      Holds actual data.
    • Method Node Class
      Method or function called by clients.
    • Object Node Class
      Real object class in real world.
      • State: Variable Node Class
      • Behavior: Method Node Class
    • ObjectType Node Class
      Real object object in real world.

    • Reference Type Node Class

      • Link Variable Node or Method Node to ObjectType Node.
      • Indicate inheritance between Nodes.

Information Model

Use XML to create specific information model.

Dirty Hexo

Hexo is a cool way to set up your own blog on GitHub despite its baffling configuration. Ready to get your hands dirty?

A. Prerequisites

  1. Git
    To check, run git -v in Git Bash
  2. Node.js
    To check, run node -v in Git Bash
  3. Markdown Knowledge

B. Hexo Configuration

Run the following commands in Git Bash.

  1. Install Hexo. Check using hexo -v
  2. Install the deployer for GitHub.
  3. This allow you to use Markdown syntax and relative path to add image to your blog.

C. Set Up Your Blog

  1. Create a folder for your blog, say gitblog.
  2. cd into gitblog.
  3. Initialize a hexo environment using hexo init.
  4. Edit _config.yml in gitblog.
  5. Generate static webpages.
  6. Local preview.
  7. Deploy.
  8. You should now be able to access your blog online on https://{yourusername}.github.io.

OPC UA: Introduction

This series are learning notes of the BRILLIANT videos Fundamentals of OPC UA by Industry40tv on YouTube.

Classic OPC

Classic OPC was a Standard Interface solving driver diversity problems using Component Object Model (COM) and Distributed Component Object Model (DCOM) to communicate between application server and client on the same and separate Windows PCs.

OPC UA

IS NOT A PROTOCOL!

4 Core Characteristics

  • Cross Platform & Internet Ready
    • Support Various Systems
    • TCP and HTTP Server/Client Communication
  • Complex Information Model
  • Service Oriented Architecture
    • Readability
    • Re-usability
    • Maintainability
  • Simplified IT Integration

2 Pillars

  • Data Modeling
    • Information Model
  • Transportation Mechanism
    • Binary Encoded Messages over TCP Protocol
    • XML Encoded Messages over HTTPS/SOAP Protocol
